# Prompt 03 - Implement With The Loop Contract

Use this only after the plan is reviewed.

```text
Implement the approved slice.

Rules:
- Keep the diff scoped.
- Do not add persistence, authentication, or a real scoring framework.
- Update verification assets with the behavior change.
- Run the cheapest relevant check first.
- If a check fails, use the exact failure as feedback.
- Stop after two failed repair attempts.

Required evidence:
- commands run and results
- Bruno API smoke evidence
- DPS-lite testautomation evidence when the loop requires heavier proof
- browser evidence or replay note if UI changed
- changed files summary
- risks and open questions
```
